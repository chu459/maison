## 第一天核心目标：搭建可用的最小可行系统

今天我们不谈复杂算法，只做三件事：

### 1. 选择正确的技术栈（避坑指南）

**错误选择**：直接上ChatGPT API → 成本失控，回复不稳定
**正确选择**：本地化模型 + 规则引擎 + GPT增强

**我的推荐组合**：
- 本地基础模型：Qwen2.5-7B（免费，中文优化）
- 规则引擎：Rasa（开源，电商场景丰富）
- GPT增强：仅用于复杂问题兜底

### 2. 配置电商专属知识库

普通客服知识库 → 回答通用问题
电商客服知识库 → **促进转化**

**必须包含的4类信息**：
1. **产品信息**：规格、材质、适用场景
2. **促销规则**：满减、优惠券、限时活动
3. **用户痛点**：尺码疑虑、物流时效、退换货政策
4. **转化话术**：如何应对“我再看看”、“太贵了”

### 3. 设计转化导向的对话流程

传统客服流程：问题 → 解答 → 结束
转化型客服流程：问题 → 解答 → **挖掘需求** → **推荐产品** → **促单**

**示例对话**：
```
用户：这件衣服适合什么场合穿？
AI：这款连衣裙采用重磅真丝材质，垂感很好，适合**商务会议、晚宴、约会**等正式场合。（解答）
AI：您是在为特定活动挑选吗？可以告诉我场合要求，我为您推荐更合适的款式。（挖掘需求）
用户：下周有个朋友婚礼
AI：婚礼场合建议选择**膝盖以下长度、避免纯白色**。我们有三款特别适合婚礼的连衣裙，其中一款新娘伴娘团采购了32件。（推荐产品）
AI：今天下单可享婚礼专属**88折优惠**，需要我为您锁定库存吗？（促单）
```
